# MuJoCo WASM — Unitree G1 Pose Viewer

A browser-based 3D simulation viewer for the **Unitree G1** humanoid robot, built on [MuJoCo WASM](https://github.com/google-deepmind/mujoco/tree/main/dist). Features real-time joint control, keyframe-based pose animation recording, and video export — all running in the browser with no backend GPU required.

---

## Features

- 🦾 Real-time joint control (grouped by limb)
- 🎬 Keyframe-based pose recording and playback
- ↓ Video export as `.webm`
- 📖 Built-in user guide with pose tips

## Quick Start — Local Development

### Prerequisites

| Tool | Version |
|------|---------|
| Node.js | ≥ 18 |
| Python | ≥ 3.10 |

### Steps

```bash
# 1. Install JS dependencies
npm install

# 2. Build the frontend bundle
npm run build

# 3. Start the HTTP server
python3 run_server.py
```

Open **http://localhost:8000** in your browser.

> **Note:** The server sets `Cross-Origin-Opener-Policy` and `Cross-Origin-Embedder-Policy` headers required by SharedArrayBuffer (used by MuJoCo WASM). A plain `python -m http.server` will **not** work.

## Deploy with Docker

### Build the image

```bash
docker build -t mujoco-viewer .
```

### Run

```bash
docker run -d \
  --name mujoco-viewer \
  -p 8000:8000 \
  mujoco-viewer
```

Open **http://localhost:8000**.

### docker-compose (recommended)

```yaml
version: "3.9"
services:
  mujoco-viewer:
    build: .
    ports:
      - "8000:8000"
    restart: unless-stopped
```

```bash
docker compose up -d
```

### With WebSocket server (optional)

The `socket_server.py` provides a WebSocket backend on port **8765** for external robot control integrations. To enable it, uncomment the relevant lines in `start.sh`:

```bash
# start.sh
python3 socket_server.py &   # WebSocket backend (port 8765)
python3 run_server.py        # HTTP frontend   (port 8000)
```

Then expose port 8765:

```bash
docker run -d -p 8000:8000 -p 8765:8765 mujoco-viewer
```

## Project Structure

```
mujoco_wasm/
├── src/
│   ├── main.js          # App entry point, Three.js scene, render loop
│   └── mujocoUtils.js   # GUI setup, keyframe recorder, help modal
├── assets/
│   └── scenes/
│       └── unitree_g1/  # Robot XML, STL meshes, textures
├── build/               # Bundled JS output (generated by npm run build)
├── run_server.py        # HTTP server with required CORP/COEP headers
├── socket_server.py     # Optional WebSocket server (port 8765)
├── Dockerfile
└── start.sh
```

## Configuration

| File | Key setting |
|------|-------------|
| `src/main.js` | `initialScene` — path to the default XML scene |
| `run_server.py` | `PORT` — HTTP port (default `8000`) |
| `socket_server.py` | `PORT` — WebSocket port (default `8765`) |

To add new scenes, place assets under `assets/scenes/<name>/` and add an entry to `assets/scenes/index.json`.

## Keyboard Shortcuts

| Key | Action |
|-----|--------|
| `Space` | Pause / Resume simulation |
| `Backspace` | Reset simulation |
| `Ctrl + L` | Reload XML scene |
| `Ctrl + A` | Reset camera |
| `K` | Save keyframe (during recording session) |
